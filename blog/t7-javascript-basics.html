January 14th - 2016

How does Javascript compare to HTML and CSS?

Javascript is a very different language to HTML and CSS because it follows different basic rules. While HTML and CSS are cascading languages (you read them from top to bottom, like your eyes cascading down the page), Javascript is an object oriented language. You (and by extension, the computer) doesn't always read it from top to bottom. In certain ways you do, it has functions that you read from top to bottom, for the most part, but not always. It's like the difference between reading a novel from start to finish (HTML, CSS) or exploring a great Where's Waldo book with your eyeballs (Javascript)

Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'

When you are writing a function, you are essentially writing a list of ordered instructions. As you know, the holidays have just been and gone, and hopefully you have thrown away your christmas tree by now. Cast your mind back though, to the night you put it up. I bet you were excited. Let's write a function for that. You might first start by naming what it is that you are doing, in english, that might be "I am putting up the christmas tree". In Javascript, you might say function PutUpChristmasTree(tree, lights) {}. The first part describes the action you are taking (The name of the function) and then the tree and its lights are also included in the process. These are called the "inputs". The function might then go on to list all the steps involved in putting up the tree, and most likely, many of these steps would also be their own functions. IE:

PutUpChristmasTree(tree, lights) {
BuyChristmasTree(money);
TakeTreeHome();
var decorations = FindDecorations(); // Here we find some more decorations to go with the light we already have.

As you can see at the end here, we have what is known as a loop. We say WHILE the god damned lights are still tangled up, untangle the god damn lights. Explain the difference between accessing data from arrays and object literals

There are many ways to store and access data in Javascript. Two different methods are as follows: Arrays. Arrays are as they sound, an array of various pieces of information, all strung together in a row. Once you put things into an array, you can access it again by way of its index. Say I added six numbers into an array. (var numbers = [4, 8, 15, 16, 23, 42]) I could retrieve the numbers by calling their position via index. It is important to note, computers start counting at 0, not 1. So if I wanted to get the fourth number form the array, I would type numbers[3]. The number in the square brackets in the index I want to retrieve. Another common way to store data is to make a new object all of your own. My children love smashing things and screaming, so lets make a fun environment in which they can smash things. In Javascript, I would define that environment like this:

var smashPalace = {
cups : 5,
expensive : ["tv", "laptop", "iphone"],
eachother : "cagematch"
};

The things in there don't really make much sense, but neither does violence. If I wanted to get some information out of this object, there are several ways I could do it. My favourite being "dot notiation". If one of my children broke a cup, I could easily update the cup=count by typing: smashPalace.cups = 4;
